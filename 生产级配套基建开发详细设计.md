# 生产级配套基建开发详细设计

## 1. 统一响应结构体设计

### 响应格式规范
```go
// 成功响应
{
    "code": 200,
    "message": "操作成功",
    "data": { ... },
    "timestamp": 1713955200,
    "request_id": "req_1234567890" // 🔺 新增：全链路追踪ID
}

// 失败响应
{
    "code": 40001,
    "message": "参数校验失败",
    "data": null,
    "timestamp": 1713955200,
    "request_id": "req_1234567890" // 🔺 新增：全链路追踪ID
}
```

### 响应码体系设计
- **2xx**: 成功类（200-299）
- **4xx**: 客户端错误类（40001-49999）
- **5xx**: 服务端错误类（50001-59999）

## 2. 全局错误码体系设计

### 错误码分类
```
├── 通用错误码 (40000-40999)
│   ├── 40000: 参数校验失败
│   ├── 40001: 请求参数缺失
│   └── 40002: 请求参数格式错误
├── 认证错误码 (41000-41999)
│   ├── 41000: 未登录
│   ├── 41001: Token过期
│   └── 41002: Token无效
├── 权限错误码 (42000-42999)
│   ├── 42000: 无权限访问
│   ├── 42001: 租户权限不足
│   └── 42002: 数据权限不足
├── 租户隔离错误码 (42100-42199) 🔺 新增
│   ├── 42100: 租户ID缺失
│   ├── 42101: 跨租户操作被禁止
│   └── 42102: 租户不存在或已禁用
└── 业务错误码 (43000-49999)
    ├── 43000: 租户不存在
    ├── 43001: 用户不存在
    └── 43002: 角色不存在
```

```go
}

// 全局异常处理中间件
func GlobalExceptionMiddleware() gin.HandlerFunc {
    return func(c *gin.Context) {
        defer func() {
            if err := recover(); err != nil {
                // 处理不同类型的异常
                handleException(c, err)
            }
        }()
        c.Next()
    }
}
```

## 6. 代码实现规范

### 统一响应结构体实现
```go
package response

// 统一响应结构
type Response struct {
    Code      int         `json:"code"`
    Message   string      `json:"message"`
    Data      interface{} `json:"data"`
    Timestamp int64       `json:"timestamp"`
}

// 成功响应
func Success(data interface{}) *Response {
    return &Response{
        Code:      200,
        Message:   "操作成功",
        Data:      data,
        Timestamp: time.Now().Unix(),
    }
}

// 失败响应
func Error(code int, message string) *Response {
    return &Response{
        Code:      code,
        Message:   message,
        Data:      nil,
        Timestamp: time.Now().Unix(),
    }
}
```

## 7. 验收标准

### 功能验收
- [ ] 接口返回格式统一，符合设计规范

### 性能验收
- [ ] 响应时间 < 100ms（正常情况）
- [ ] 异常处理不影响正常请求
- [ ] 日志写入不影响接口性能

- [ ] 错误信息不泄露系统内部细节
- [ ] 配置信息加密存储
