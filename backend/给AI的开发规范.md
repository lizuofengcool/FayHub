# 给 AI 的 FayHub 项目开发强制规范

## 核心铁律（违反直接回滚）
1. **绝对禁止使用 `utils.GetGlobalDB()`，所有数据库操作必须用 `utils.GetDB(ctx)`**
2. **绝对禁止在 Controller 里直接操作数据库，必须调用 Service 层**
3. **绝对禁止使用旧响应工具 `utils.Ok*`/`utils.Fail*`，必须统一用 `response.Gin*` 系列方法**
4. **所有租户ID必须从 Context 中获取，绝对禁止从前端传参中读取 `tenant_id`**
5. **所有注释、错误提示、接口返回信息必须使用中文，禁止英文注释（技术术语除外）**

## 分层规范
- **Controller**：参数校验 → 调用Service → 返回统一响应，无业务逻辑
- **Service**：业务逻辑处理 → 调用 `utils.GetDB(ctx)` 做数据操作，无HTTP相关逻辑
- **Model**：纯数据结构定义，无业务逻辑
- **Repository**：数据访问层，仅做数据CRUD封装

## AI开发要求
1. **先输出技术实现方案，确认后再写完整代码**
2. **每次只开发一个功能点，禁止一次性修改多个文件/多个模块**
3. **写完代码必须先运行项目根目录的 `检查脚本.bat`，确保预检通过**
4. **绝对不能重复实现项目中已经存在的工具函数、基础方法**
5. **禁止修改项目核心基类（`BaseModel`/`TenantModel`）、核心配置、目录结构**
