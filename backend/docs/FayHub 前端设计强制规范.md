# FayHub 前端设计强制规范（给AI的铁律）
## 核心定位
FayHub 是企业级多租户SaaS底座，界面必须是**专业、简洁、大气、高效的B端产品风格**，禁止花里胡哨、土味、杂乱的设计，严格遵循Element Plus官方设计规范。

## 设计铁律（违反直接重写）
1.  **配色规范**
    - 主色调：固定使用 Element Plus 官方品牌色 #409EFF（极客蓝），辅助色仅可使用 #67C23A（成功）、#E6A23C（警告）、#F56C6C（危险）、#909399（信息）
    - 禁止使用任何非主流配色、渐变背景、花里胡哨的色块，背景仅可使用 #FFFFFF、#F5F7FA 两个色值
    - 文字配色严格遵循层级：标题 #303133、正文 #606266、次要信息 #909399、禁用信息 #C0C4CC

2.  **布局规范**
    - 后台固定使用「左侧菜单+顶部导航+主内容区」经典三栏布局，禁止花里胡哨的布局样式
    - 内容区严格使用卡片式布局，卡片间距固定20px，内边距固定24px，禁止随意调整
    - 所有页面必须有清晰的视觉层级，标题区、操作区、内容区、分页区严格分离，禁止杂乱堆砌

3.  **组件规范**
    - 100%使用 Element Plus 原生组件，禁止自定义组件样式、禁止重写组件原生交互，仅可调整组件尺寸
    - 表单页：标签左对齐，输入框宽度统一360px，表单间距固定22px，操作按钮固定在底部左对齐
    - 列表页：表格带斑马纹，操作列固定在最右侧，批量操作按钮在表格左上角，搜索区在表格上方，分页在表格右下角
    - 按钮规范：主操作按钮用primary类型，次要操作用default类型，危险操作用danger类型，禁止一个页面出现多个主按钮

4.  **字体与间距规范**
    - 字体统一使用「思源黑体/微软雅黑」，禁止使用任何艺术字体
    - 字体大小层级：页面大标题20px加粗、卡片标题16px加粗、正文14px、次要信息12px
    - 所有间距必须是4px的倍数，禁止出现奇数间距，保证视觉一致性

5.  **交互与动画规范**
    - 仅保留Element Plus组件原生动画，禁止添加任何自定义动画、悬浮特效、弹窗动效
    - 所有操作必须有明确的反馈：成功/失败提示、加载状态、禁用状态
    - 禁止任何影响操作效率的冗余交互，B端产品以高效、易用为第一原则

## AI开发要求
1.  先输出静态HTML+CSS代码，仅使用Element Plus CDN引入，不涉及Vue、业务逻辑、接口调用
2.  严格遵循上述设计规范，禁止任何偏离
3.  代码必须整洁、注释清晰，可直接在浏览器打开运行
4.  响应式适配：最小适配1366*768分辨率，最大适配1920*1080分辨率
