# FayHub 用户界面文本规范

## 📋 概述

本文档定义了 FayHub 项目中所有用户界面文本的规范和标准，确保界面文本的一致性和专业性。

## 🎯 基本原则

### 1. 用户友好原则
- **清晰易懂**：使用简单明了的语言，避免技术术语
- **积极正面**：使用积极的语言，避免负面词汇
- **指导性强**：提供明确的指导和操作建议

### 2. 一致性原则
- **术语统一**：相同概念使用相同术语
- **风格一致**：保持语言风格的一致性
- **格式规范**：遵循统一的文本格式

### 3. 专业性原则
- **准确无误**：确保文本内容准确无误
- **简洁明了**：避免冗长和复杂的表达
- **符合规范**：遵循行业标准和最佳实践

## 📝 文本分类规范

### 1. 按钮文本规范

#### 操作按钮
- **创建/新增**："创建用户"、"新增租户"
- **编辑/修改**："编辑信息"、"修改配置"
- **删除/移除**："删除用户"、"移除角色"
- **保存/提交**："保存更改"、"提交审核"
- **取消/关闭**："取消操作"、"关闭窗口"

#### 状态按钮
- **启用/激活**："启用账号"、"激活租户"
- **禁用/停用**："禁用账号"、"停用服务"
- **重置/恢复**："重置密码"、"恢复默认"

### 2. 表单标签规范

#### 必填字段
- 使用星号(*)标记必填字段
- 标签文本简洁明了："用户名*"、"邮箱地址*"

#### 可选字段
- 不添加特殊标记
- 可添加说明文本："备注（选填）"

#### 字段说明
- 使用简洁的说明文本
- 提供格式要求："密码（8-20位字符）"

### 3. 提示信息规范

#### 成功提示
- **格式**：操作名称 + "成功"
- **示例**："用户创建成功"、"密码修改成功"

#### 错误提示
- **格式**：问题描述 + 解决方案
- **示例**："用户名已存在，请使用其他用户名"

#### 警告提示
- **格式**：风险提示 + 确认要求
- **示例**："此操作不可逆，确定要删除吗？"

#### 加载提示
- **格式**：操作名称 + "中..."
- **示例**："数据加载中..."、"正在保存..."

### 4. 页面标题规范

#### 列表页面
- **格式**：模块名称 + "列表"
- **示例**："用户列表"、"租户列表"

#### 详情页面
- **格式**：模块名称 + "详情"
- **示例**："用户详情"、"租户详情"

#### 操作页面
- **格式**：操作名称 + 模块名称
- **示例**："创建用户"、"编辑租户"

## 🎨 文本格式规范

### 1. 标点符号使用
- **句号**：陈述句末尾使用句号
- **问号**：疑问句末尾使用问号
- **感叹号**：谨慎使用，避免过度使用
- **冒号**：标签和内容之间使用冒号
- **括号**：补充说明使用括号

### 2. 数字和单位
- **数字**：使用阿拉伯数字
- **单位**：数字和单位之间留空格
- **示例**："有效期 30 天"、"文件大小 2 MB"

### 3. 日期和时间
- **日期格式**："YYYY年MM月DD日"
- **时间格式**："HH:mm:ss"
- **示例**："2024年01月15日 14:30:00"

### 4. 特殊字符
- **星号**：必填字段标记
- **省略号**：表示加载或未完成状态
- **箭头**：表示导航或方向

## 📚 文本维护和更新

### 维护流程
1. **需求分析**：分析文本修改需求
2. **文本设计**：设计符合规范的文本
3. **评审确认**：团队评审文本内容
4. **实施更新**：更新相关界面文本
5. **测试验证**：验证文本显示效果

### 更新原则
- **及时性**：发现问题时及时更新
- **一致性**：确保更新后文本一致性
- **完整性**：更新所有相关位置的文本
